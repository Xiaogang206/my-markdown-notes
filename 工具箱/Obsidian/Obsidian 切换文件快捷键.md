Obsidian 核心本身没有直接提供“上一个/下一个同文件夹文件”的内置命令。

Neighbouring Files 插件本身**没有默认快捷键**，但它提供了两个核心命令，你需要在 Obsidian 的热键设置里手动为它们绑定快捷键。

### 步骤（Mac OS 下操作）

1. 确保插件已安装并启用  
   - 社区插件 → 浏览 → 搜索 "Neighbouring Files" → 安装 & 启用  
   （插件 ID：neighbouring-files，作者 Fabian Untermoser）

2. 打开热键设置  
   - 按快捷键：`⌘ ,` （<font color="#ff0000">Command + 逗号</font>）  
   - 或者：左下角齿轮图标 → 设置 → 热键（Hotkeys）

3. 搜索命令  
   在<font color="#ff0000">热键设置界面</font>的搜索框输入以下关键词之一（推荐直接复制）：  
   - `neighbouring`  
   - `next file`  
   - `prev file`

   你会看到两个命令（英文名称通常是这样的）：

   - **Neighbouring Files: Navigate to next file**  
     （导航到下一个文件）

   - **Neighbouring Files: Navigate to prev file**  
     （导航到上一个文件）

   有些版本也可能显示为：
   - Navigate to next file
   - Navigate to previous file

4. 为它们设置你喜欢的快捷键（Mac 常用推荐）

   | 功能           | 推荐快捷键（Mac）          | 说明                              |
   |----------------|--------|--------------|
   | 下一个文件     | `Ctrl + Shift + .`         | 很多人用这个（. 在键盘右下角好按） |
   | 上一个文件     | `Ctrl + Shift + ,`         | , 和 . 是一对，很容易记住          |
   | 备选方案1      | `⌘ + Option + →` / `←`     | 类似浏览器前进后退的感觉           |
   | 备选方案2      | `Ctrl + →` / `Ctrl + ←`    | 更简单，但可能和系统冲突           |
   | 备选方案3      | `⌘ + Shift + →` / `←`      | 更强力组合，避免误触               |

   点击命令右侧的空白区域 → 按下你想要的按键组合 → 确认无冲突（如果显示冲突就换一个）。

5. 测试  
   打开同一个文件夹内的多个笔记，用你刚设的快捷键切换，应该能顺畅上下跳动。  
   排序依据：默认按**文件名字母顺序**，也可以在插件设置里开启「使用 File Explorer 的排序方式」或「反向排序」。

最受欢迎的组合（很多中文用户也在用）就是：

- 下一个：**Ctrl + Shift + .**  
- 上一个：**Ctrl + Shift + ,**

这样设置完就和很多人习惯一致了，祝使用愉快！如果遇到冲突或命令没出现，欢迎补充描述，我再帮你排查。
