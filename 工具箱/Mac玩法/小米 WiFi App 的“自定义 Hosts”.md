在小米 WiFi App 的“自定义 Hosts”功能中，其实质是将你的**小米路由器**变成了一个局域网内的 **DNS 解析服务器**。

作为开发者，你可以将其理解为：在路由器层挂载了一个**全局 Hosts 文件**。

---

## 1. 核心作用：全家桶式屏蔽与重定向

你之前在 macOS 和 Windows 上分别修改 Hosts，只能影响那台电脑。而在路由器插件里修改，**所有连接该 WiFi 的设备**（手机、平板、电视、IoT 设备）都会同时生效。

### A. 彻底屏蔽广告和追踪

很多智能电视或手机 App 的广告插件会向特定的域名请求数据。

- **操作**：将广告域名指向 `127.0.0.1`。
    
- **结果**：电视开机广告、App 内置弹窗由于解析不到真正的服务器，直接失效。
    

### B. 局域网设备“域名化” (开发者最爱)

如果你家里有 Synology NAS 或跑着 Docker 的服务器，每次访问都要输入 `192.168.31.100:8080` 非常麻烦。

- **操作**：设置 `nas.com` 指向 `192.168.31.100`。
    
- **结果**：你只需要在浏览器输入 `nas.com:8080` 就能访问，不再需要记 IP。
    

### C. 强制重定向

你可以将某个想戒掉的网站定向到百度或者你自己的本地服务器。

- **操作**：`www.v2ex.com` 指向 `127.0.0.1`。
    

---

## 2. 相比电脑端修改的优势

- **移动端克星**：iPhone 和未 Root 的安卓手机很难直接修改系统 Hosts。通过路由器修改，是这类设备实现域名屏蔽的唯一简便手段。
    
- **一次配置，全家受益**：不需要在每台电脑、每台手机上重复操作。
    

---

## 3. 一针见血的警示：为什么有时候“没卵用”？

如果你在小米 App 里设了 Hosts 却发现没生效，通常是以下三个原因：

1. **加密 DNS (DoH/DoT)**：
    
    现在的 Chrome、Safari 以及最新的安卓/iOS 系统都有“加密 DNS”功能。如果设备开启了该功能，它会绕过路由器的 53 端口（DNS 端口）直接和外部服务器通信，**路由器 Hosts 拦截就会完全失效**。
    
2. **浏览器缓存**：
    
    设备会优先读取自己内存里的 DNS 缓存。你改完路由器后，手机可能还在用旧的记录。
    
    - **对策**：手机开关一次飞行模式，或者重启 WiFi。
        
3. **App 硬编码 IP**：
    
    有些极其流氓的 App 不通过 DNS 查询域名，而是直接把服务器 IP 写死在代码里，这种情况下 Hosts 拿它没办法。

## 4. Obsidian 笔记：路由器 Hosts 配置流

### 路由器层 Hosts 应用场景
- **拦截广告**: 屏蔽特定 API 域名。
- **内网映射**: 
    - `obsidian.pc` -> `192.168.31.xx` (方便同步同步)
    - `nas.local` -> `192.168.31.yy`
- **家长控制**: 屏蔽特定游戏或短视频服务器。

### 失效排查清单
1. 检查设备是否开启了 "私有 DNS" 或 "安全 DNS"。
2. 检查设备是否挂着代理 (VPN/Clash)。
3. 尝试重启路由器 DNS 服务或重新连接 WiFi。
