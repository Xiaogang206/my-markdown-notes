# Quartz + Vercel 个人博客搭建与维护教程

## 一、 核心架构方案

- **内容管理 (Obsidian)：** 所有的笔记编写、双链组织都在本地 Obsidian 完成。
- **代码仓库 (GitHub)：** 存储 Quartz 框架源码和 Markdown 笔记。
- **自动化部署 (Vercel)：** 只要 GitHub 收到更新，Vercel 会自动拉取并重新编译生成网页。

## 二、 初始部署步骤

### 1. 准备仓库

- 确保 Quartz 项目（包含 `quartz.config.ts`、`content/` 文件夹等）已推送到 GitHub。
- **关键：** `content/` 文件夹内必须包含 `index.md`，否则首页会报 404 错误。

### 2. Vercel 托管配置

1. 登录 Vercel，通过 GitHub 账号登录。
2. 点击 **"Add New..." -> "Project"**，导入你的 Quartz 仓库。
3. **构建设置 (Build Settings)：**
    - **Framework Preset:** 选择 **Other**。
    - **Build Command:** 输入 `npx quartz build`。
    - **Output Directory:** 输入 `public`。
4. **环境设置 (Settings -> General)：**
    - **Node.js Version:** 确保选择 **20.x** 或更高版本。
    - 点击 **Deploy** 等待编译完成。

## 三、 域名与访问配置

### 1. 设置自定义域名

- 在 Vercel 项目页面的 **Settings -> Domains** 中，可以输入你喜欢的名字（如 `xiaogang206.vercel.app`）并点击 **Add**。
- Vercel 默认生成的长域名（带 `git-v4` 等字符）是预览域名，仅用于测试。

### 2. 同步 `baseUrl` (非常重要)

1. 拿到定好的域名后，回到本地修改 `quartz.config.ts`。
2. 找到 `baseUrl` 字段，填入你的域名（例如 `xiaogang206.vercel.app`）。
3. **注意：** 格式要求**不带** `https://`，且末尾**不带**斜杠 `/`。
4. 修改后执行 `git push`，让 Vercel 重新按正确地址编译。

## 四、 日常维护工作流

1. **写作：** 在 Obsidian 的 `content` 文件夹里新建或修改 Markdown 文件。
2. **管理：** 文件夹结构会自动反映在网页左侧的 Explorer 目录树中。
3. **发布：** 使用 Git 执行“三部曲”：
    - `git add .`
    - `git commit -m "更新内容描述"`
    - `git push`
4. **生效：** 推送后，Vercel 会在 1-2 分钟内自动更新网页。

## 五、 进阶小技巧与排雷

- **修改首页标题：** 打开 `index.md`，修改顶部的 YAML 区域中的 `title` 字段，即可更改浏览器标签页显示的标题。
- **图片处理：** 建议图片命名使用英文或数字，避免中文命名导致部署后链接失效。
- **Node.js 版本警告：** 如果收到关于 `engines` 的警告，可以在 `package.json` 中将 `"node": ">=22"` 改为 `"node": "22.x"` 来锁死版本。
- **404 报错排查：** 如果看到 404，通常是因为 Vercel 还在编译中，或者 `Build Command` 写错了。可以去 Vercel 的 **Deployments** 查看 **Build Logs** 了解具体原因。
