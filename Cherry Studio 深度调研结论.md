# Cherry Studio 深度调研结论

## 1. 核心定位

**Cherry Studio** 是一款**本地优先**的多模型 AI 桌面客户端。它本身不提供 AI 能力，而是通过 API Key（云端）或 Ollama（本地）连接各种大语言模型，并集成本地知识库功能。

## 2. 硬件匹配：Mac M4

- **版本选择**：必须下载 **`arm64`** 版本（或通过 Homebrew 安装）。
- **性能优势**：M4 芯片的神经网络引擎可极速处理本地向量化计算。
- **安装命令**：`brew install --cask cherry-studio`

## 3. 本地知识库 (RAG) 机制

| **维度**     | **结论**                                                     |
| ------------ | ------------------------------------------------------------ |
| **容量限制** | **不受限**。仅受限于你的 Mac 硬盘容量，无文件数量限制。      |
| **隐私安全** | **可控**。本地索引不上传；但若使用云端 API (如 DeepSeek)，检索到的资料片段会发送给服务商。 |
| **绝对隐私** | 建议配合 **Ollama** 运行本地模型，实现 100% 离线数据闭环。   |
| **文件格式** | 完美支持 **Markdown **、PDF、Docx 等。                       |

## 4. 关键优势与风险

### ✅ 优势

- **数据主权**：所有对话历史和知识库索引均加密存储在本地。
- **零成本扩展**：无需订阅费，按需支付 API 流量费或使用纯免费本地模型。
- **工作流集成**：支持 MCP 协议和智能体插件，扩展性强。

### ❌ 风险与限制

- **同步局限**：作为本地软件，缺乏原生跨设备同步，需手动备份数据。
- **Token 消耗**：知识库开启时，单次提问会附带大量背景资料，增加云端 API 成本。
- **OCR 依赖**：对于扫描件 PDF 识别率有限，建议优先使用文本类资料。

## 拆解“本地知识库”工作的两个关键环节：

### 1. 数据的“拆解”阶段：本地索引（安全）

当你把 PDF 或 Markdown 文件导入 Cherry Studio 构建知识库时，软件会在**你的本地电脑上**完成以下操作：

- **读取与切片**：将文件拆分成小的文本块。
- **向量化存储**：将这些文本块转换成数字（向量），并存储在本地的向量数据库中。
- **结论**：这个过程完全在你的 Mac M4 芯片上运行，**不涉及云端**。

### 2. AI 的“阅读”阶段：检索与回答（风险点）

当你向 AI 提问时，逻辑如下：

1. Cherry Studio 在本地库里搜到相关的“资料片段”。
2. 它会将这些“资料片段”连同你的问题，一起打包发送给 AI 模型。
3. **重点就在这里**：
   - **如果你使用在线 API**（如 DeepSeek、OpenAI、Claude）：你的资料片段**会发送到这些公司的服务器**。虽然它们声称 API 数据不会用于训练，但技术上数据确实离开了一次你的电脑。
   - **如果你使用本地模型**（如通过 Ollama 运行 Llama 3 或 DeepSeek）：数据流转**完全闭环在你的 Mac 上**。没有任何字节会流向互联网。

### 如何实现“绝对隐私”？

如果你对隐私有极高要求，建议按以下方案配置：

| **方案**                            | **隐私级别** | **性能要求**             | **说明**                                               |
| ----------------------------------- | ------------ | ------------------------ | ------------------------------------------------------ |
| **Cherry Studio + API (云端)**      | 中等         | 低                       | 适合处理非敏感资料。数据会传给 AI 公司，但不会被公开。 |
| **Cherry Studio + Ollama (全本地)** | **最高**     | **高 (M4 性能绰绰有余)** | 在本地运行模型。资料**永远不会**离开你的电脑。         |

------

### 一针见血的建议

1. **别被“本地”两个字完全迷惑**：只要你在设置里填了 API Key（比如 SiliconFlow 或 DeepSeek 的 Key），你问问题时，AI 就在“读”你的私有资料。
2. **善用 M4 的性能**：你手里的 M4 芯片拥有极强的神经网络引擎（Neural Engine），完全建议你安装 **Ollama**，然后在 Cherry Studio 里接入本地模型。
3. **敏感数据物理隔离**：如果是公司机密、财务报表或极度隐私的个人日记，**严禁**在连接了云端 API 的状态下进行询问。

**总结：** 软件（Cherry Studio）不偷你资料，但如果你把资料喂给云端 AI（如 ChatGPT），那是你自己把资料“送”出去的。
