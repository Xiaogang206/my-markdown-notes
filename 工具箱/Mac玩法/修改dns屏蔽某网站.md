### 1. 手动输入的格式规范

在 `hosts` 文件中，每一行代表一个指向关系。格式必须是：

**`IP地址` + `空格/制表符` + `域名`**

Plaintext

```
127.0.0.1 www.badsite.com
127.0.0.1 badsite.com
```

- **一针见血的提醒**：如果你想屏蔽一个网站，建议把 `www` 开头和不带 `www` 的域名都写上。有些网站（如 Bilibili）有大量的子域名（如 `live.bilibili.com`），如果你不手动加上这些子域名，只屏蔽主站是没用的。
    

---

### 2. 如何在 Nano 中自由移动和输入

- **移动光标**：使用键盘上的 **方向键**（上下左右）。鼠标点击在终端里通常是无效的。
    
- **换行**：移动到文件末尾，直接按 **Enter** 键即可开始新的一行。
    
- **删除**：使用 **Backspace**（退格键）删除光标前的字符。
    
- **注释**：如果你想暂时取消屏蔽但不想删掉这行字，可以在行首加一个 `#` 号。
    
    - 例如：`# 127.0.0.1 www.example.com`（这一行会被系统忽略）。
        

---

### 3. 操作全流程（经王志刚验证有效）

1. **打开**：`sudo nano /etc/hosts`
    
2. **定位**：按方向键下，直到光标到达文件最后一行。
    
3. **输入**：按回车换行，输入 `127.0.0.1 目标网址`。
    
4. **保存 (WriteOut)**：按下 `Ctrl + O`，接着按 `Enter`。
    
5. **退出 (Exit)**：按下 `Ctrl + X`。
    
6. **刷新 DNS**：执行 `sudo killall -HUP mDNSResponder`。（大小写要正确才行）

### 4. 为什么有时候手动加入了还是无效？

如果手动输入并保存后，Chrome 依然能打开网页，通常是以下两个原因：

- **浏览器缓存**：Chrome 会缓存 DNS。尝试在地址栏输入 `chrome://net-internals/#dns` 然后点击 **Clear host cache**。
    
- **DoH (DNS over HTTPS)**：这是 Chrome 的一个特性，它会绕过系统的 hosts 文件直接去查 DNS。
    
    - **解决方法**：去 Chrome **设置 -> 隐私和安全 -> 安全**，关闭“使用安全 DNS”选项。
