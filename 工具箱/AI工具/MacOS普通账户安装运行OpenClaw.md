## **🦞 为什么“普通账户”安装小龙虾更合适？**

来自多篇官方与社区教程都强调：

- **绝对不要用 root 账户运行 OpenClaw**  
    （风险极高）知乎专栏
- **推荐使用普通用户账户部署**  
    ，避免 AI 工具获得系统级写入权限，从而保护系统安全知乎专栏

### **✔ 普通账户的优势**

- **权限受限**  
    ：即使小龙虾被误导执行危险操作，也无法修改系统文件。
- **更安全**  
    ：降低误删系统文件、覆盖配置、破坏系统的风险。
- **官方推荐**  
    ：macOS 原生适配，普通账户即可顺利安装运行，无需管理员权限参与大部分流程。知乎专栏 +3

## **🔐 为什么不能用管理员账户？**

因为小龙虾（OpenClaw）属于 **高权限 AI Agent**，它可以：

- 访问本地文件
- 执行命令
- 自动化任务

如果用管理员账户运行：

- AI 可能被误导执行危险操作
- 可能修改系统配置
- 甚至可能删除关键文件

因此官方明确写了：

> **“绝对不要用 root 账户直接部署运行。”**

## **🧭 最佳实践：如何在普通账户中安全安装？**

下面是最推荐的方式（来自多篇教程整合）：

### **1. 在普通账户打开终端**

无需切换管理员。

### **2. 安装 Homebrew（如未安装）**

教程中提供了完整步骤

### **3. 安装 Node.js ≥ 22**

可用 brew 或 nvm 安装

### **4. 一键安装 OpenClaw**

官方命令（macOS/Linux）：

```
curl -fsSL https://openclaw.ai/install.sh | bash
```

### **5. 运行配置向导**

```
openclaw onboard
```
