**MCP** 的全称是 **Model Context Protocol（模型上下文协议）**，由 Anthropic 公司在 2024 年 11 月 25 日发布。它是一套**统一的工具接入标准**，旨在让大语言模型能够更方便、更标准地使用各类外部工具和感知外部环境。

以下是关于 MCP 的核心理解：

### 1. 核心定位：AI 界的“Type-C”接口

在 MCP 出现之前，如果你想让同一个工具（如天气查询、文件读取）在不同的 AI 平台（如 ChatGPT、Claude、Gemini）上运行，通常需要按照每个平台的规范编写多套接入代码。**MCP 的出现统一了这一标准**，开发者只需按 MCP 规范开发一次工具，该工具就可以被所有支持 MCP 的宿主软件（Host）直接使用，极大地提高了通用性。

### 2. 系统架构中的三个角色

整个 MCP 的运行链路通常涉及以下三个角色：

- **大模型 (Model)**：负责理解用户意图、选择合适的工具并对结果进行归纳总结。
- **MCP Host (宿主/客户端)**：支持 MCP 协议的软件，如 **Cursor**、**Claude Desktop**、**Cline** 或 **Cherry Studio**。它扮演“翻译官”和“调度员”的角色，负责将工具列表传给模型，并根据模型指令去调用工具。
- **MCP Server (服务器)**：本质上是一个符合 MCP 规范的**程序**（通常由 Python 或 Node 编写）。它内部封装了多个 **Tools（工具，本质是函数）**，负责执行具体任务并返回数据。

### 3. MCP 能做什么？

通过接入 MCP，大模型可以突破其知识库的时间限制，拥有“感官和四肢”：

- **获取实时信息**：如查询天气、股票行情或实时路况。
- **操作外部软件**：如操作 Unity 编写游戏、抓取网页内容。
- **连接本地资源**：如读取本地文件、搜索代码定义或访问特定的数据库。

### 4. 关键技术细节

- **通信方式**：MCP Server 与 Client 之间主要通过 **stdio（标准输入输出）** 或 **SSE（服务器发送事件）** 进行沟通。
- **Tool 的定义**：Server 会通过 **JSON Schema** 规范来描述工具的功能和参数要求（input schema），以便模型准确提取参数并调用。
- **安全性与可控性**：相比于直接运行命令行（CLI），MCP 采用结构化的数据传输（如 JSON），更加**安全可控**，且不容易因特殊字符（如引号）导致执行报错。

### 5. 与其他概念的区别

- **与 Function Calling 的关系**：两者是**互补**而非替代关系。MCP 作用于宿主与函数之间，规定如何发现和驱动工具；而 Function Calling 是模型层面的能力，规定了模型如何挑选工具和解析结果。
- **与 Agent Skill 的区别**：**MCP 负责提供数据**（连接外部世界），而 **Agent Skill 负责教模型如何处理这些数据**（提供做事规矩和指令文档）。
